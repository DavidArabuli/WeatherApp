=========================================================================

Plans:

Izvēloties PHP un PostgreSQL kā pamatu, izveidot tīmekļa aplikāciju ar vienkāršotu MVC arhitektūru.

Controller - atbildēs par lietotāja pieprasījumu apstrādi un datu sagatavošanu skatam.

Model - nodrošinās darbu ar PostgreSQL datubāzi (pilsētu saraksts).

View - lietotāja interfeiss (galvenā lapa ar pilsētu laikapstākļiem un detalizēts skats).

Papildus: serviss API darbam.

0) Iepazīties ar OpenWeatherMap API dokumentāciju, notestēt API atslēgu un savienojumu ar API.

1) Izveidot galveno lapu, kurā tiek attēloti laikapstākļi vismaz 10 pilsētām visā pasaulē (sākotnēji "hardcoded").

2) Dot iespēju lietotājam apskatīt padziļinātu informāciju par konkrētas pilsētas laikapstākļiem, realizējot to:

- vai nu kā atsevišķu lapu,

- vai kā izvēršamu skatu, nospiežot uz pilsētas.

3) Ieviest iespēju pārslēgties starp mērvienībām, izmantojot vienkāršu “toggle” pogu.

4) Uzstādīt un konfigurēt PostgreSQL, izveidot sākotnējo datubāzes tabulu pilsētu sarakstam.

4.2) Dot iespēju:

pievienot jaunas pilsētas,

dzēst esošās pilsētas no saraksta.

Datubāzē tiks glabāta tikai minimāli nepieciešamā informācija (piemēram, pilsētas nosaukums, valsts kods, ģeogrāfiskais platums un garums), lai pēc tam varētu dinamiski iegūt aktuālos laikapstākļu datus no API.

5) Pievienot CSS stilus, lai aplikācija būtu vizuāli pievilcīga.

6) GitHub repozitorijs + readme.

7) Aplikāciju izvietot uz publiski pieejama VPS hostinga, nodrošinot piekļuvi caur publisku URL. 

8) Sagatavot Docker Compose konfigurāciju ar PHP un PostgreSQL servisiem.
